Создай веб-приложение Sportpath с использованием MAX UI библиотеки компонентов. Это мобильное приложение для бронирования спортивных площадок.

**Шаг 1: Установка и настройка MAX UI**
1. Установи библиотеку: `npm install @maxhub/max-ui`
2. Импортируй стили в главный файл: `import '@maxhub/max-ui/dist/styles.css'`
3. Оберни всё приложение в провайдер MaxUI:
```jsx
import { MaxUI } from '@maxhub/max-ui';

function App() {
  return (
    <MaxUI platform="ios" colorScheme="light">
      {/* тут будут все компоненты приложения */}
    </MaxUI>
  );
}
Используй полиморфные компоненты через asChild prop для гибкости
Шаг 2: Структура приложения и компоненты MAX UI
Используй следующие компоненты из библиотеки:

Для хедера: Panel с mode="secondary" + Flex для расположения кнопок
Для карты: Container с кастомным стилем для Yandex Maps
Для карточек площадок: Card или Panel с Typography.Title, Avatar, Badge
Для кнопок: Button с разными mode (primary для основных действий)
Для профиля: используй компоненты из раздела Profile compositions
Для списка записей: List и ListItem
Для рейтинга: Icon с звездочками или кастомный компонент
Для bottom sheet: Sheet или Panel с position="bottom"
Шаг 3: Экраны приложения

1. Главный экран (карта)

В хедере слева: Avatar с фото пользователя (размер 32px)
В хедере справа: Button с иконкой фильтра
Над картой: погода на 5 дней (горизонтальный скролл с Card компонентами)
Карта Yandex Maps (просто placeholder для мокапа)
На карте: маркеры спортивных площадок (используй кастомные иконки)
При клике на маркер: выезжающий снизу Panel с информацией о площадке:
Фото площадки (Avatar.Image в большом размере)
Название (Typography.Title)
Рейтинг (5 звездочек)
Краткое описание
Статус: "Сегодня 16:00, 5 человек"
Кнопки: "Записаться" (primary), "Построить маршрут" (secondary)
2. Экран профиля

Шапка: Avatar (размер 72px) + имя пользователя (Typography.Title)
Раздел "Активные записи":
Список с иконками погоды (тучка/солнышко) для каждого бронирования
Дата, время, название площадки
Кнопка "Построить маршрут"
Раздел "История записей":
Похожий список, но без кнопок маршрута
3. Экран фильтров

Modal или отдельный экран с фильтрами по типам площадок (футбол, баскетбол и т.д.)

Шаг 5: Требования к стилю

Приложение должно выглядеть как нативное iOS приложение (platform="ios")
Используй светлую тему (colorScheme="light")
Для иконок используй компоненты из MAX UI или кастомные SVG
Все компоненты должны быть адаптированы под мобильные экраны (320px - 414px)
Анимации при переходах между экранами и при открытии bottom sheet
Шрифты и отступы должны соответствовать дизайн-системе MAX UI
Шаг 6: Особые указания

Карта Yandex Maps должна быть реализована как placeholder (просто div с фоновым цветом и надписью "Yandex Maps")
Все переходы между экранами делай через условный рендеринг в одном компоненте App
Используй React hooks (useState, useEffect) для управления состоянием
Для маршрутизации не нужна библиотека, достаточно простого state с текущим экраном
Все стили должны быть через CSS-in-JS или кастомные классы с использованием innerClassNames prop из MAX UI
Приложение должно быть в едином стиле MAX UI - используй их компоненты вместо нативных HTML элементов где это возможно
Шаг 7: Финальная проверка
Убедись, что:

Все компоненты MAX UI правильно импортированы
Стили подключены
Приложение выглядит как мобильное iOS приложение
Есть все описанные экраны и функционал
Используются мок-данные для демонстрации
Приложение полностью работает на мок-данных без бэкенда
